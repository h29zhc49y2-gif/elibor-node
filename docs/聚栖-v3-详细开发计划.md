## 一、项目概述

### 1.1 产品定位

| 项目 | 说明 |
|------|------|
| **产品名称** | Elibor · 聚栖计划 |
| **产品类型** | AI栖人养成模拟游戏（文字+像素风格） |
| **核心玩法** | 玩家作为观察者，观察栖人在星球上的自主生活，可进行有限干预 |
| **目标用户** | 喜欢模拟经营、养成类游戏的玩家；对AI行为模拟感兴趣的科技爱好者；追求轻松休闲、碎片化时间游戏的用户 |

### 1.2 世界观设定

- **时间背景**：2XXX年人类向宇宙发射搭载克隆体的飞行器
- **核心场景**：栖人在荒凉星球上生存、建设、发展
- **生命周期**：儿童期 → 青年期 → 成年期 → 老年期 → 暮年期 → 死亡

### 1.3 核心系统

| 系统 | 描述 |
|------|------|
| AI行为引擎 | 需求驱动、记忆系统、个性系统（OCEAN五维人格） |
| 五维指数系统 | 工业、农业、居住、科技、能源五个维度 |
| 干预系统 | 有限但有意义的玩家互动 |
| 传记系统 | 从出生到死亡的完整叙事体验 |
| 每日播报 | 动物森友会风格的友好界面 |

## 四、Phase 7 — 每日播报系统

### 4.1 核心概念

**"每日播报"**：用户首次登录时自动弹出，展示星球上过去"一天"（现实时间2.4小时）发生的大事和用户栖人的活动记录。

**时间换算**：
- 6分钟 = 星球1小时
- 144分钟（2.4小时）= 星球1天
- 显示格式："第127天"

### 4.2 播报内容生成规则

#### 4.2.1 星球动态

| 内容类型 | 触发条件 | 示例 |
|----------|----------|------|
| 指数突破 | 任意指数达到阈值 | "工业指数突破80！" |
| 新发现 | 栖人发现新资源 | "发现新矿脉！" |
| 技术突破 | 科技指数提升 | "能源技术取得突破！" |
| 新栖人加入 | 有新用户创建栖人 | "欢迎新成员加入！" |
| 栖人离世 | 有栖人死亡 | "阿木安详离世，享年68岁" |
| 随机事件 | 概率触发 | "今天天气晴朗，适合工作" |

#### 4.2.2 我的栖人动态

| 内容类型 | 记录依据 | 示例 |
|----------|----------|------|
| 工作记录 | 工作时长和产出 | "工作8小时，开采50单位矿石" |
| 社交记录 | 社交互动 | "与小月进行了愉快的交谈" |
| 成就获取 | 成就系统 | "获得'勤劳矿工'成就" |
| 状态变化 | 需求系统 | "今天状态良好，效率提升" |
| 特殊事件 | 随机触发 | "发现了一块罕见的能量水晶" |

## 九、Phase 9 — 技术栈迁移（Railway + Node.js）

### 9.1 迁移背景

**当前问题**：
- Hostinger 共享主机 `.htaccess` 路由规则不生效
- 无法支持定时任务（引擎自动运算的核心需求）
- 无法自定义 Apache 配置
- SSH 需要密码认证，无法自动化部署

**聚栖核心需求**：
- 引擎必须 **24小时常驻运行**，不能依赖用户请求触发
- 每 **6分钟** 执行一次行为决策（星球1小时）
- 栖人状态持续更新，需要可靠的数据持久化
- 支持并发处理大量栖人运算

### 9.2 技术栈选择

| 层级 | 技术 | 说明 |
|------|------|------|
| **运行时** | Node.js 20+ | TypeScript 优先，异步处理能力强 |
| **框架** | Express / Fastify | 轻量 HTTP 框架 |
| **ORM** | Prisma | 类型安全，支持 PostgreSQL |
| **数据库** | PostgreSQL | Railway 自带，功能强大 |
| **缓存** | Redis | 可选，用于会话和排行榜缓存 |
| **部署** | Railway | 支持常驻 Worker + Cron Jobs |
| **实时** | Socket.IO | WebSocket 推送 |
| **进程管理** | PM2 | 多进程管理（本地开发） |

### 9.5 引擎 Worker 实现

```typescript
// src/engine/worker.ts
import { CronJob } from 'cron';
import { PrismaClient } from '@prisma/client';
import { TimeEngine } from './time-engine';
import { BehaviorEngine } from './behavior-engine';
import { EconomyEngine } from './economy-engine';
import { EventEngine } from './event-engine';

const prisma = new PrismaClient();
const timeEngine = new TimeEngine(prisma);
const behaviorEngine = new BehaviorEngine(prisma);
const economyEngine = new EconomyEngine(prisma);
const eventEngine = new EventEngine(prisma);

async function engineTick() {
    const startTime = Date.now();
    console.log(`[Engine] Tick started at ${new Date().toISOString()}`);

    try {
        // 1. 更新时间
        const currentTime = await timeEngine.advance();

        // 2. 获取所有活跃栖人
        const activeSouls = await prisma.soul.findMany({
            where: { status: 'alive' }
        });

        // 3. 批量处理栖人行为
        for (const soul of activeSouls) {
            // 需求衰减
            await behaviorEngine.updateNeeds(soul);

            // 行为决策（OCEAN个性 + 记忆 + 需求）
            const action = await behaviorEngine.decideAction(soul);

            // 执行行为
            await behaviorEngine.executeAction(soul, action);

            // 经济产出
            await economyEngine.processProduction(soul, action);
        }

        // 4. 更新星球指数
        await economyEngine.updatePlanetStats(activeSouls);

        // 5. 处理随机事件
        await eventEngine.processRandomEvents(activeSouls);

        // 6. 处理社交事件
        await eventEngine.processSocialEvents(activeSouls);

        const duration = Date.now() - startTime;
        console.log(`[Engine] Tick completed in ${duration}ms, processed ${activeSouls.length} souls`);
    } catch (error) {
        console.error('[Engine] Tick failed:', error);
    }
}

// 每6分钟执行一次（星球1小时）
const job = new CronJob('*/6 * * * *', engineTick);

job.start();
console.log('[Engine] Worker started, running every 6 minutes...');
```
