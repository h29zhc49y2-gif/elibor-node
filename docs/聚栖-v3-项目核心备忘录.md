## 一、世界观设定

### 1.1 背景故事
- **时代**：2XXX年，人类文明扩张期
- **计划名称**：聚栖计划（Elibor Project）
- **核心目标**：向宇宙播种生命，建立星际殖民地网络
- **执行方式**：发射搭载克隆体（栖人）的无人飞行器，随机降落在宜居星球

### 1.2 栖人设定
- **形态**：统一外表，128×128像素头像呈现
- **生命周期**：儿童期 → 中年期 → 老年期 → 暮年期 → 死亡
- **死亡处理**：生成个人传记，入驻荣誉堂
- **玩家选择**：死亡后可重新选择新栖人

### 1.3 目标星球
- **类型**：自然环境恶劣，但矿产资源丰富
- **环境特点**：风暴频繁、昼夜温差大、地表崎岖
- **资源优势**：稀有矿物、能源晶体、特殊材料

## 七、核心算法

### 7.1 贡献值计算
```
贡献值 = 职业效率 × 工作时长 × 情绪加成 × 随机波动(0.8-1.2)
```

### 7.2 干预采纳概率
```
采纳概率 = 基础概率(50%) × 心情加成 × 个性匹配度 × 需求匹配度 × 随机因子
```

### 7.3 行为决策算法
```
行为选择概率 = 
  (需求优先级 × 需求紧迫度) + 
  (记忆权重 × 相关度) + 
  (个性倾向 × 行为匹配度) + 
  随机因子(0-0.1)
```

## 十、技术方案评估（2026-05-28）

### 10.1 当前问题

Hostinger 共享主机存在以下限制：
1. `.htaccess` 重写规则不生效
2. 无法自定义 Apache 配置
3. SSH 需要密码认证，无法自动化
4. Hostinger API 不包含数据库管理功能

### 10.2 引擎自动运算需求

聚栖项目有复杂的引擎逻辑需要考虑：
- **时间引擎**：6分钟 = 星球1小时，144分钟 = 星球1天
- **生活引擎**：栖人行动、消耗、产出
- **经济系统**：资源产出、建造
- **每日播报**：自动生成新闻
- **定时任务**：定期执行引擎逻辑

### 10.3 推荐方案

| 方案 | 后端 | 数据库 | 定时任务 | 免费额度 | 适合度 |
|------|------|--------|----------|----------|--------|
| **① Railway** | Node.js/Python | PostgreSQL | ✅ Cron | 5刀/月 | ⭐⭐⭐⭐⭐ |
| **② Render** | Node.js/Python | PostgreSQL | ✅ Cron | 750小时/月 | ⭐⭐⭐⭐ |
| **③ Fly.io** | Docker | SQLite/PostgreSQL | ✅ Cron | 3个免费实例 | ⭐⭐⭐⭐ |
| **④ Cloudflare Workers + D1** | Workers | D1 (SQLite) | ✅ Cron Triggers | 10万次/天 | ⭐⭐⭐ |

### 10.4 最终推荐：Railway + PostgreSQL
