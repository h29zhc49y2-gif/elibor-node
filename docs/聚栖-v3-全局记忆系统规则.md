# 聚栖 v3 — 全局记忆系统规则

> 文档版本：v1.0
> 创建日期：2026-05-28
> 适用范围：栖人记忆系统的核心规则，防止记忆中断

---

## 一、记忆系统架构

### 1.1 记忆层次结构

```
记忆系统
├── 短时记忆（Short-Term Memory, STM）
│   └── 容量：最多10条记录
│   └── 保存时长：最多24小时（星球时间）
│   └── 用途：当前行为决策参考
├── 长时记忆（Long-Term Memory, LTM）
│   ├── 成就记忆：里程碑事件、职业成就
│   ├── 人际关系记忆：与其他栖人的关系
│   ├── 技能记忆：掌握的技能、知识
│   └── 创伤记忆：重大负面事件
└── 永久记忆（Permanent Memory）
    └── 初始个性：创建时分配的OCEAN属性
    └── 生涯总览：生平统计数据
```

### 1.2 记忆数据结构

**短时记忆条目格式**：
```json
{
  "id": "stm_xxx",
  "type": "work|social|learning|rest|event",
  "content": "完成采矿工作，获得10单位矿石",
  "timestamp": "2026-05-28 10:30:00",
  "location": "矿区A",
  "related_souls": ["soul_001"],
  "emotion_impact": 5
}
```

**长时记忆格式**：
```json
{
  "achievements": [
    {"id": "ach_001", "name": "首次采矿", "timestamp": "2026-05-28 09:00", "impact": 10}
  ],
  "relationships": {
    "soul_002": {"intimacy": 85, "last_meet": "2026-05-28 14:00", "interaction_count": 23}
  },
  "skills": {"mining": 75, "farming": 30},
  "traumas": [{"type": "accident", "timestamp": "2026-05-20", "severity": 60}]
}
```

## 二、记忆存储规则

### 2.1 短时记忆存储规则

| 规则编号 | 规则描述 | 触发条件 | 执行动作 |
|----------|----------|----------|----------|
| STM-001 | 行为结束后立即记录 | 任何行为完成时 | 将行为记录添加到短时记忆 |
| STM-002 | 容量限制 | 短时记忆达到10条 | 删除最旧的1条记录 |
| STM-003 | 时间限制 | 记录超过24小时（星球时间） | 自动转移到长时记忆或删除 |
| STM-004 | 情绪阈值过滤 | 情绪影响 < 3 | 不记录到短时记忆 |
| STM-005 | 重复行为合并 | 相同行为连续发生 | 更新计数，不新增记录 |

### 2.2 长时记忆存储规则

| 规则编号 | 规则描述 | 触发条件 | 执行动作 |
|----------|----------|----------|----------|
| LTM-001 | 成就记录 | 完成里程碑事件 | 添加到achievements数组 |
| LTM-002 | 关系更新 | 与其他栖人互动 | 更新relationships中的亲密度 |
| LTM-003 | 技能增长 | 完成学习行为 | 更新skills中的熟练度 |
| LTM-004 | 创伤记录 | 遭遇重大负面事件 | 添加到traumas数组 |
| LTM-005 | 短时转长时 | 短时记忆条目超过24小时且有意义 | 转移到长时记忆 |

### 2.3 永久记忆保护规则

| 规则编号 | 规则描述 | 保护内容 | 保护措施 |
|----------|----------|----------|----------|
| PERM-001 | 个性不变原则 | OCEAN五维度基础值 | 仅允许±5%的波动 |
| PERM-002 | 生涯数据累计 | 总贡献值、工作时长等 | 只能增加，不能减少 |
| PERM-003 | 死亡传记保护 | 死亡时生成的传记 | 永久保存，不可修改 |

## 三、记忆检索规则

### 3.1 检索优先级规则

| 优先级 | 记忆类型 | 权重 | 检索场景 |
|--------|----------|------|----------|
| 1 | 短时记忆（最近1小时） | 1.0 | 当前行为决策 |
| 2 | 短时记忆（1-6小时） | 0.8 | 近期行为参考 |
| 3 | 长时记忆-技能 | 0.7 | 工作选择决策 |
| 4 | 长时记忆-人际关系 | 0.6 | 社交行为决策 |
| 5 | 长时记忆-成就 | 0.5 | 自信度计算 |
| 6 | 长时记忆-创伤 | 0.4 | 恐惧/回避行为 |
| 7 | 永久记忆-个性 | 0.3 | 行为倾向判断 |

### 3.2 场景匹配规则

| 场景类型 | 检索关键词 | 记忆匹配策略 |
|----------|------------|--------------|
| 工作场景 | 职业、技能、工具、产出 | 优先匹配技能记忆 |
| 社交场景 | 其他栖人ID、亲密度、互动历史 | 优先匹配人际关系记忆 |
| 休息场景 | 疲劳值、睡眠、放松 | 优先匹配最近休息记录 |
| 学习场景 | 知识、技能、学习效率 | 优先匹配技能记忆和学习历史 |
| 危机场景 | 危险、威胁、伤害 | 优先匹配创伤记忆和安全记录 |

### 3.3 记忆关联规则

| 规则编号 | 规则描述 | 示例 |
|----------|----------|------|
| RET-001 | 时间邻近关联 | 同一时间段的事件优先关联 |
| RET-002 | 地点关联 | 同一地点的记忆优先检索 |
| RET-003 | 人物关联 | 涉及同一栖人的记忆优先检索 |
| RET-004 | 情绪关联 | 相似情绪状态下的记忆优先检索 |
| RET-005 | 因果关联 | 有因果关系的记忆串联检索 |

## 四、记忆更新规则

### 4.1 个性影响规则

| 个性维度 | 记忆影响 | 更新方式 |
|----------|----------|----------|
| 外向性(O) | 社交记忆权重增加 | 社交记录权重×(1 + O/200) |
| 尽责性(C) | 工作记忆权重增加 | 工作记录权重×(1 + C/200) |
| 情绪稳定性(E) | 创伤记忆影响降低 | 创伤权重×(1 - E/200) |
| 宜人性(A) | 人际关系记忆增强 | 关系记录权重×(1 + A/200) |
| 开放性(N) | 学习记忆权重增加 | 学习记录权重×(1 + N/200) |

### 4.2 记忆衰退规则

| 记忆类型 | 衰退速率 | 计算公式 |
|----------|----------|----------|
| 短时记忆 | 快速衰退 | 权重 = 初始权重 × (1 - 时间差/86400) |
| 长时记忆-技能 | 慢速衰退 | 权重 = 初始权重 × (1 - 时间差/864000) |
| 长时记忆-人际关系 | 中等衰退 | 权重 = 初始权重 × (1 - 时间差/259200) |
| 长时记忆-成就 | 极慢衰退 | 权重 = 初始权重 × (1 - 时间差/31536000) |
| 长时记忆-创伤 | 缓慢衰退 | 权重 = 初始权重 × (1 - 时间差/1296000) |

### 4.3 记忆增强规则

| 规则编号 | 规则描述 | 增强条件 | 增强效果 |
|----------|----------|----------|----------|
| ENH-001 | 重复强化 | 相同记忆被多次检索 | 权重×1.1（最多×2） |
| ENH-002 | 情绪强化 | 记忆伴随强烈情绪 | 权重×1.2 |
| ENH-003 | 成就强化 | 记忆与成就相关 | 权重×1.3 |
| ENH-004 | 社交强化 | 记忆涉及多人互动 | 权重×1.15 |

## 五、记忆清理规则

### 5.1 短时记忆清理

| 规则编号 | 清理条件 | 处理方式 |
|----------|----------|----------|
| CLEAN-001 | 记录超过24小时且情绪影响<5 | 直接删除 |
| CLEAN-002 | 记录超过24小时且情绪影响≥5 | 转移到长时记忆 |
| CLEAN-003 | 容量超过10条 | 删除最旧且权重最低的记录 |
| CLEAN-004 | 完全重复的记录 | 合并计数，保留最新时间 |

### 5.2 长时记忆清理

| 规则编号 | 清理条件 | 处理方式 |
|----------|----------|----------|
| CLEAN-005 | 技能记忆权重<0.1且超过30天未使用 | 标记为"遗忘"状态 |
| CLEAN-006 | 人际关系记忆亲密度<10且超过14天未互动 | 标记为"疏远"状态 |
| CLEAN-007 | 创伤记忆权重<0.1 | 标记为"愈合"状态 |
| CLEAN-008 | 成就记忆永不删除 | 永久保留 |

### 5.3 清理执行策略

| 执行时机 | 执行频率 | 清理范围 |
|----------|----------|----------|
| 栖人行为完成后 | 每次 | 检查短时记忆容量 |
| 星球时间每天凌晨 | 每日 | 全面清理过期记忆 |
| 记忆检索失败时 | 按需 | 检查记忆完整性 |
| 栖人状态保存时 | 每5分钟 | 检查并优化记忆结构 |

## 六、记忆与行为关联规则

### 6.1 行为决策中的记忆应用

| 行为类型 | 参考记忆类型 | 决策权重 |
|----------|--------------|----------|
| 工作选择 | 技能记忆、成就记忆 | 0.6 |
| 社交选择 | 人际关系记忆、社交历史 | 0.7 |
| 休息选择 | 疲劳记忆、睡眠历史 | 0.8 |
| 学习选择 | 技能记忆、学习历史 | 0.7 |
| 危机应对 | 创伤记忆、安全记录 | 0.9 |

### 6.2 记忆驱动的行为模式

| 模式名称 | 触发条件 | 行为表现 |
|----------|----------|----------|
| 习惯行为 | 相同场景重复≥5次 | 自动选择熟悉行为 |
| 回避行为 | 创伤记忆触发 | 远离相关场景/人物 |
| 追求行为 | 成就记忆触发 | 主动追求相似成就 |
| 社交偏好 | 亲密关系记忆 | 优先选择亲密栖人互动 |

### 6.3 记忆中断处理

| 中断类型 | 检测条件 | 恢复策略 |
|----------|----------|----------|
| 短时记忆丢失 | STM记录为0或异常 | 从长时记忆重建最近行为 |
| 长时记忆损坏 | LTM数据不完整 | 标记损坏部分，逐步恢复 |
| 记忆冲突 | 多条记忆矛盾 | 按时间优先和情绪强度判定 |
| 记忆过载 | 记忆数量超过阈值 | 启动紧急清理流程 |

## 七、记忆持久化规则

### 7.1 保存时机

| 时机 | 触发条件 | 保存内容 |
|------|----------|----------|
| 实时保存 | 每次记忆更新 | 增量保存变化部分 |
| 定时保存 | 每5分钟 | 完整状态快照 |
| 事件保存 | 重大事件发生时 | 事件相关记忆完整保存 |
| 周期保存 | 每日凌晨 | 完整备份 |

### 7.2 备份策略

| 备份类型 | 频率 | 保留期限 | 存储位置 |
|----------|------|----------|----------|
| 增量备份 | 每小时 | 7天 | 本地备份目录 |
| 完整备份 | 每日 | 30天 | 远程存储 |
| 灾难恢复备份 | 每周 | 永久 | 离线存储 |

### 7.3 恢复优先级

| 优先级 | 恢复内容 | 恢复顺序 |
|--------|----------|----------|
| 1 | 永久记忆（个性、生涯数据） | 最先恢复 |
| 2 | 长时记忆（成就、技能） | 其次恢复 |
| 3 | 长时记忆（人际关系、创伤） | 第三恢复 |
| 4 | 短时记忆 | 最后恢复（可重建） |
