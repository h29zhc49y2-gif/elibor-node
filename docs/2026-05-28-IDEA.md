## 一、技术栈迁移方案

### 1.1 问题背景

聚栖是全自动算法驱动的模拟养成系统，与普通用户交互型游戏不同：
- 引擎必须 **24小时常驻运行**
- 每 **6分钟** 执行一次行为决策
- 栖人状态持续更新
- 用户只是"观察者"

当前 Hostinger 方案的问题：
- `.htaccess` 路由规则不生效
- 无法支持定时任务（引擎自动运算的核心需求）
- 无法自定义 Apache 配置
- SSH 需要密码认证，无法自动化部署

### 1.2 选择方案：Railway + Node.js + PostgreSQL

**理由**：
1. 支持常驻 Worker 进程 — 引擎可24小时运行
2. 支持 Cron Jobs — 每6分钟执行一次
3. PostgreSQL 事务保证数据一致性
4. $5/月，免费额度 500 小时/月
5. 自动部署，连接 GitHub

### 1.3 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| 运行时 | Node.js 20+ | TypeScript 优先 |
| 框架 | Express | 轻量 HTTP 框架 |
| ORM | Prisma | 类型安全，支持 PostgreSQL |
| 数据库 | PostgreSQL | Railway 自带 |
| 部署 | Railway | 支持常驻 Worker + Cron Jobs |
| 实时 | Socket.IO | WebSocket 推送 |

## 二、引擎架构

### 2.1 核心设计

每 6 分钟执行一次（星球 1 小时）：

```
┌─────────────────────────────────────────────────────────────┐
│                    引擎 Worker 架构                         │
├─────────────────────────────────────────────────────────────┤
│  每6分钟执行一次（星球1小时）                                │
│  │                                                          │
│  ├─ 1. 时间引擎：更新星球时间                                │
│  ├─ 2. 行为引擎：栖人自主决策                                │
│  │     ├─ 需求衰减（饥饿、精力、社交、学习、情绪）            │
│  │     ├─ 行为决策（OCEAN个性 + 记忆 + 需求）                │
│  │     └─ 执行行为 + 更新状态                                │
│  ├─ 3. 经济引擎：资源产出 + 星球指数更新                      │
│  ├─ 4. 事件引擎：随机事件 + 社交事件                         │
│  └─ 5. 播报引擎：每日播报生成                                │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 行为引擎算法

**OCEAN 个性模型**：
- Openness（开放性）：影响探索、学习意愿
- Conscientiousness（尽责性）：影响工作积极性
- Extraversion（外向性）：影响社交活动
- Agreeableness（宜人性）：影响人际关系
- Neuroticism（神经质）：影响情绪稳定性

**行为决策流程**：
1. 计算每个需求的紧迫度（0-100）
2. 结合 OCEAN 个性权重，计算各行为得分
3. 使用加权随机选择最终行为
4. 根据行为类型确定产出和位置
