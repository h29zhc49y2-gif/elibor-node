## 一、记忆层次结构

### 1.1 记忆层级定义

| 层级 | 名称 | 存储时长 | 容量限制 | 优先级 |
|------|------|----------|----------|--------|
| L1 | 瞬时记忆 | 当前对话 | 无限（会话内） | 最高 |
| L2 | 短期记忆 | 24小时 | 最近100条关键信息 | 高 |
| L3 | 长期记忆 | 项目周期 | 结构化知识库 | 中 |
| L4 | 永久记忆 | 永久 | 核心文档链接 | 低 |

### 1.2 记忆内容分类

| 类别 | 内容示例 | 存储层级 |
|------|----------|----------|
| 项目状态 | 当前阶段、已完成任务 | L1+L2+L3 |
| 设计决策 | 架构选择、技术方案 | L2+L3+L4 |
| 代码结构 | 文件位置、API端点 | L3+L4 |
| 业务规则 | 世界观、系统规则 | L3+L4 |
| 待办事项 | 开发计划、任务清单 | L1+L2 |
| 对话历史 | 用户指令、决策记录 | L1+L2 |

## 二、记忆存储规则

### 2.1 瞬时记忆（L1）规则

| 规则编号 | 规则描述 | 触发条件 |
|----------|----------|----------|
| MEM-L1-001 | 实时记忆当前对话 | 每轮对话自动记录 |
| MEM-L1-002 | 记忆对话上下文 | 保持完整对话历史 |
| MEM-L1-003 | 追踪当前任务状态 | 实时更新任务进度 |
| MEM-L1-004 | 保留用户偏好 | 记录用户习惯和偏好 |

### 2.2 短期记忆（L2）规则

| 规则编号 | 规则描述 | 触发条件 |
|----------|----------|----------|
| MEM-L2-001 | 每日总结记录 | 每日结束时自动生成 |
| MEM-L2-002 | 关键决策快照 | 重要决策后立即记录 |
| MEM-L2-003 | 待办事项追踪 | 任务状态变化时更新 |
| MEM-L2-004 | 冲突检测 | 发现矛盾信息时标记 |
| MEM-L2-005 | 24小时清理 | 超过24小时自动归档 |

### 2.3 长期记忆（L3）规则

| 规则编号 | 规则描述 | 触发条件 |
|----------|----------|----------|
| MEM-L3-001 | 文档链接存储 | 重要文档创建后记录 |
| MEM-L3-002 | 架构图存储 | 系统设计完成后记录 |
| MEM-L3-003 | API清单存储 | API定义完成后记录 |
| MEM-L3-004 | 业务规则存储 | 规则制定后记录 |
| MEM-L3-005 | 定期回顾 | 每周检查完整性 |

### 2.4 永久记忆（L4）规则

| 规则编号 | 规则描述 | 触发条件 |
|----------|----------|----------|
| MEM-L4-001 | 核心文档永久保存 | 项目基石文档 |
| MEM-L4-002 | 设计决策永久保存 | 重大架构决策 |
| MEM-L4-003 | 永不删除 | 除非明确废弃 |

## 三、记忆检索规则

### 3.1 检索优先级

| 优先级 | 记忆层级 | 检索顺序 |
|--------|----------|----------|
| 1 | L1 - 瞬时记忆 | 首先检索 |
| 2 | L2 - 短期记忆 | 其次检索 |
| 3 | L3 - 长期记忆 | 再次检索 |
| 4 | L4 - 永久记忆 | 最后检索 |

### 3.2 检索策略

| 场景 | 检索重点 |
|------|----------|
| 代码实现 | 文件位置、API端点、架构设计 |
| 设计讨论 | 业务规则、世界观设定、系统规则 |
| 任务追踪 | 开发计划、任务状态、待办事项 |
| 问题排查 | 错误日志、历史修复、已知问题 |

### 3.3 关联检索规则

| 规则编号 | 规则描述 | 示例 |
|----------|----------|------|
| RET-001 | 上下文关联 | 用户提到"栖人"时检索相关规则 |
| RET-002 | 时间关联 | 最近修改的文档优先 |
| RET-003 | 主题关联 | 同一主题的信息一起检索 |
| RET-004 | 层级关联 | 从具体到抽象逐层检索 |

## 四、记忆更新规则

### 4.1 更新时机

| 时机 | 更新内容 | 更新层级 |
|------|----------|----------|
| 文档创建 | 记录文档位置和摘要 | L3+L4 |
| 代码修改 | 记录修改内容和原因 | L2+L3 |
| 决策达成 | 记录决策内容和依据 | L2+L3+L4 |
| 任务完成 | 更新任务状态 | L1+L2 |
| 错误修复 | 记录错误原因和解决方案 | L2+L3 |

### 4.2 更新策略

| 规则编号 | 规则描述 | 执行方式 |
|----------|----------|----------|
| UPD-001 | 实时更新L1 | 每次对话后更新 |
| UPD-002 | 定时更新L2 | 每小时批量更新 |
| UPD-003 | 事件驱动更新L3 | 重要事件发生时更新 |
| UPD-004 | 手动确认更新L4 | 需要用户确认后更新 |

## 五、记忆维护规则

### 5.1 定期检查

| 检查类型 | 频率 | 检查内容 |
|----------|------|----------|
| 每日检查 | 每天结束 | L2记忆完整性 |
| 每周检查 | 每周结束 | L3记忆一致性 |
| 每月检查 | 每月结束 | L4记忆准确性 |

### 5.2 冲突处理

| 冲突类型 | 处理策略 |
|----------|----------|
| 信息矛盾 | 标记并提示用户确认 |
| 过时信息 | 标记为"待更新" |
| 重复信息 | 合并并去重 |
| 缺失信息 | 标记为"待补充" |

### 5.3 清理策略

| 记忆层级 | 清理条件 | 处理方式 |
|----------|----------|----------|
| L1 | 对话结束 | 自动清除 |
| L2 | 超过24小时 | 归档到L3 |
| L3 | 超过30天未访问 | 标记为"不活跃" |
| L4 | 永不清理 | 永久保留 |

## 六、记忆恢复规则

### 6.1 恢复优先级

| 优先级 | 恢复内容 | 恢复来源 |
|--------|----------|----------|
| 1 | 核心文档位置 | L4永久记忆 |
| 2 | 当前任务状态 | L2短期记忆 |
| 3 | 项目架构设计 | L3长期记忆 |
| 4 | 对话历史 | L2短期记忆 |

### 6.2 恢复流程

```
记忆中断检测 → 加载L4永久记忆 → 加载L3长期记忆 → 加载L2短期记忆 → 重建L1瞬时记忆
```

### 6.3 恢复检查清单

| 检查项 | 验证内容 |
|--------|----------|
| 核心文档 | 所有关键文档可访问 |
| 项目状态 | 当前阶段和任务状态正确 |
| 设计决策 | 所有重大决策记录完整 |
| 代码结构 | 文件位置和API端点正确 |

## 八、记忆检查与验证

### 8.1 自我检查清单

每次对话前自动执行：

1. ✅ 检查L4永久记忆完整性
2. ✅ 检查当前项目阶段状态
3. ✅ 检查待办事项列表
4. ✅ 检查是否有未解决的冲突
5. ✅ 检查文档链接有效性

### 8.2 用户确认机制

| 场景 | 确认方式 |
|------|----------|
| 重大决策 | 复述决策内容请用户确认 |
| 任务完成 | 总结完成内容请用户确认 |
| 冲突解决 | 列出矛盾点请用户选择 |
| 计划变更 | 说明变更原因请用户批准 |

### 8.3 记忆刷新机制

| 触发条件 | 刷新内容 |
|----------|----------|
| 对话中断超过1小时 | 重新加载L2-L4记忆 |
| 用户询问"记忆"相关问题 | 执行完整记忆检查 |
| 检测到信息矛盾 | 触发冲突解决流程 |
